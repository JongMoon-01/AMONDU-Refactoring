## 현재 진행 중인 작업: ERD 최적화 리팩터링

### 발견된 문제 3가지

[문제 1] User 테이블 중복
- edtech.User: userId Long (PK), email, name, passwordHash, phone
- attention.User: id Integer (PK), username, email, is_active — edtech User와 완전히 별개, 동기화 없음

→ 해결 방향: attention.User 테이블 삭제
  - Session.user_id, Score.user_id, UserStats.user_id 타입을 Long으로 변경
  - JWT에서 edtech userId 추출해서 직접 사용
  - /api/score/users/ 엔드포인트 전체 제거
  - 이 작업이 마지막 대화에서 진행 중이었음 (코드 리팩터링 단계에서 토큰 소진)

[문제 2] Summary 테이블 중복
- edtech.Summary: summaryId Long, lecture_id FK, content TEXT
- aiquiz.Summary: summary_id BigInt, lecture_id (참조만), content TEXT — 같은 데이터 중복, 정합성 위험

→ 해결 방향: aiquiz.Summary 삭제
  - aiquiz는 edtech.Summary.summaryId만 참조
  - REST 조회 또는 Kafka 이벤트로 수신

[문제 3] CourseEngagementAnalytics.attentionArr JSON 배열
- 현재: 집중도 구간 데이터를 JSON 배열로 통째로 저장 → 조회 비효율, 실시간 처리 불가

→ 해결 방향: Kafka 연동 후
  - attentionArr 컬럼 제거
  - avgFocusScore: Float, focusDropCount: Int 집계 컬럼으로 교체


## 전체 ERD 테이블 목록

### edtech DB (MariaDB)
- User: userId(Long PK), email(unique), name, passwordHash, phone
- ClassEntity: classId(Long PK), title, tag, headcount
- Enrollment: id(Long PK), user_id FK→User, class_id FK→ClassEntity, createdAt
- Course: courseId(Long PK), title, materialUrl, tag, instructor, lecture_id FK→Lecture, class_id FK→ClassEntity
- Lecture: lectureId(Long PK), title, mpdPath, vttPath
- Summary: summaryId(Long PK), lecture_id FK→Lecture, content TEXT, createdAt
- CourseEngagementAnalytics: courseAnalyticsId(Long PK), class_id FK→ClassEntity, courseId, userId(String), startedAt/endedAt(Instant), totalDurationSec, attentionArr(JSON[]) ← 문제 컬럼, createdAt

### noticeboard DB (MariaDB)
- NoticePost: id(Long PK), title, content TEXT, createdAt/updatedAt
- QnaPost: id(Long PK), title, content TEXT, authorId(String), createdAt/updatedAt
- QnaComment: id(Long PK), qna_post_id FK→QnaPost, content TEXT

### focus_analysis DB (MySQL) — attention-model-fastapi-service
- User: id(Integer PK), username, email, is_active, created_at ← 삭제 예정
- Session: id(Integer PK), user_id FK→User, session_name, start_time/end_time, duration_minutes, total_frames, is_active
- Score: id(Integer PK), user_id FK→User, session_id FK→Session, emotion_score/gaze_score/final_score(Float), timestamp
- FrameData: id(Integer PK), session_id FK→Session, frame_id, emotion/gaze data

### aiquiz DB (MySQL)
- (Summary 있음 — 삭제 예정)
- AIQuiz 등 (quiz 관련 테이블)
